## 문제

```c
int main() {
    long long long long long long
    long long long long long long
    long long long long long long
    long long long long long long
    long long long long long long
```

오늘은 혜아의 면접 날이다. 면접 준비를 열심히 해서 앞선 질문들을 잘 대답한 혜아는 이제 마지막으로 칠판에 직접 코딩하는 문제를 받았다.<br>
혜아가 받은 문제는 두 수를 더하는 문제였다. C++ 책을 열심히 읽었던 혜아는 간단히 두 수를 더하는 코드를 칠판에 적었다.<br>
코드를 본 면접관은 다음 질문을 했다. “만약, 입출력이 $N$바이트 크기의 정수라면 프로그램을 어떻게 구현해야 할까요?”

혜아는 책에 있는 정수 자료형과 관련된 내용을 기억해 냈다. 책에는 `long int`는 $4$바이트 정수까지 저장할 수 있는 정수 자료형이고 `long long int`는 
$8$바이트 정수까지 저장할 수 있는 정수 자료형이라고 적혀 있었다.<br>
혜아는 이런 생각이 들었다. “`int` 앞에 `long`을 하나씩 더 붙일 때마다 $4$바이트씩 저장할 수 있는 공간이 늘어나는 걸까?<br>
분명 `long long long int`는 $12$바이트, `long long long long int`는 $16$바이트까지 저장할 수 있는 정수 자료형일 거야!” <br>
그렇게 혜아는 당황하는 면접관의 얼굴을 뒤로한 채 칠판에 정수 자료형을 써 내려가기 시작했다.

혜아가 $N$바이트 정수까지 저장할 수 있다고 생각해서 칠판에 쓴 정수 자료형의 이름은 무엇일까?

___

## 입력

첫 번째 줄에는 문제의 정수 $N$이 주어진다. $(4\le N\le 1\, 000$; $N$은 $4$의 배수$)$ 
___

## 출력

혜아가 $N$바이트 정수까지 저장할 수 있다고 생각하는 정수 자료형의 이름을 출력하여라.
___

## 예제 입력 & 출력 1

| 예제 입력 | 예제 출력 |
|-------|-----|
| 4   | long int |

---

## 예제 입력 & 출력 2

| 예제 입력 | 예제 출력 |
|-------|-----|
| 20   | long long long long long int |

---

## 해결방안
**1. 입력받은 바이트 수 $N$을 4로 나눈 몫만큼 반복하여 `long` 문자열을 생성하고, 마지막에 `int`를 붙여 완성된 문장을 출력한다.**
